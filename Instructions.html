MittensEdit is a web-based Javascript AA editor based on OrinrinEditor and built atop the Codemirror library.

The editor is currently still in early development, there are more features to come. Back up your AA regularly while working in case of bugs. Mostly tested in Firefox so far.

※ AA Editor requires that canvas and webfonts be enabled for correct dot width calculations.
Editor will not function correctly for characters outside the Saitamaar webfont. Do not use tab characters for spacing.

A Note on Fonts:

AA, or S_JIS art, on 2ch was always made for display using MS PGothic's 16px bitmaps. Since modern displays are such that CSS pixels do not always match physical pixels, and some modern systems do not display bitmap fonts, viewing AA with MS PGothic as originally intended can be difficult. To work around this, MittensEdit uses the webfont Saitamaar. Saitamaar is based on the version of MS PGothic's 16px bitmaps that was available in the Windows XP era. This will display the overwhelming majority of AA correctly.

The term "S_JIS art" is somewhat of a misnomer. The font is far more essential than the encoding and plenty of AA uses Unicode characters when available in MS PGothic's bitmaps.

The only caveats are that certain changes made in MS PGothic's bitmaps around Windows 7 changed the visual appearance (but thankfully not the metrics) of certain glyphs. This means that Saitamaar displays early AA perfectly while certain, more recent AA will display not as intended. You are unlikely to notice this difference. Additionally, certain Unicode characters, which are rarely found in some AA, are omitted in Saitamaar. There are plans to produce an updated version of Saitamaar in the future that adds these missing glyphs.

Some systems will see "color fringing" with Saitamaar due to subpixel positioning. An attempt to correct this will be made in the future.

More information on AA fonts and their history may be found in the AA Bar's AA Info Megathread (COMING SOON)

A guide to AA spaces:

‣ Full-Width Space (FWS) - An ideographic space, (U+3000). 11px wide. Does not collapse when repeated.
‣ Half-Width Space (HWS) - A normal ASCII space, (U+0020). 5px wide. Collapses when multiple are used in a row, or when one occurs at the start of a line.

AA uses a mixture of FWS and HWS for padding empty areas. Due to multiple HWS in a row collapsing into one HWS when posted, and any amount of HWSs at the start of a line collapsing into nothing, MittensEdit marks these spaces red. If you attempt to post an AA with these invalid spaces on most boards, it will not display correctly and you will look like an idiot.

Some AA uses a collection of non-SJIS "Unicode spaces" for achieving paddings that cannot be achieved with just HWSs and FWSs. MittensEdit and OrinrinEditor both introduce unicode spaces where necessary to achieve exact padding. The generation of unicode spaces can be toggled with the "Use Unicode Spaces" switch in the menu bar, or with Alt-U. Many AA craftsmen attempt to minimize or entirely eliminate unicode spaces before posting their AA.

Do not attempt to enter any sort of whitespace character (especially tab characters) into the editor which are not among those the editor generates naturally or among those which are available in the manual space insertion dropdown. The editor will not handle these other whitespace characters correctly and there is no guarantee that they will render identically in all possible environments your AA might be viewed. It is highly recommended that you enter HWSs as normal with your keyboard when padding and then use editor commands (Ctrl-LeftArrow, Ctrl-RightArrow) to acheive the desired correct spacing.

All AA you will find in the wild will use nothing more than FWS, HWS, and occasionally the handful of Unicode spaces that OrinrinEditor and MittensEdit use, it is vanishingly unlikely that you will encounter AA with unusual whitespace that MittensEdit cannot handle.

It is also recommended that you leave Unicode space generation enabled while carrying out most editor functions. This will ensure that every operation is carried out in as close to a pixel-perfect manner as is possible. Eliminating Unicode spaces should be carried out after large editing operations, but do keep in mind that, if you wish to eliminate Unicode spaces, you should probably plan ahead for it. Some editor operations will not function as expected when Unicode space generation is disabled.

Editor Operations:

※Layerboxes

The main purpose of an AA editor.
You can insert a piece of AA as a semi-transparent "layerbox", move it to a target location, and paste it down. It will be composited into the document as if you were pasting images. The editor will attempt to preserve the original formatting as closely as possible.
If Unicode space generation is enabled, this should be pixel-perfect. If it is not, something about how fonts are displaying on your device or how your browser handles HTML canvas is misconfigured.
If Unicode space gen is disabled, the editor will attempt to keep the formatting as close as possible, possibly using periods for padding.

A Layerbox has four buttons:

- Paste layerbox directly

- "Adjustable paste", pastes the AA in a read-only preview. Use your arrow keys to adjust the position before finalizing with Enter. Identical to pasting behavior when Paste Mode is set to Block Pasting Enabled.

- Edit Layerbox, allows you to edit the Layerbox's contents. Behaves as a normal text area, AA editor functions will not apply to it with the exception of 1px space adjustment with Ctrl-LeftArrow & Ctrl-RightArrow.

- Closes the Layerbox.

By default, a Layerbox will be deleted after pasting. This cannot be undone. To not delete a layerbox after pasting (ie. for pasting the same AA multiple times), there is a "Delete Layerbox After Paste" setting in the Settings panel.

When compositing, spaces "outside" a line in the Layerbox's contents will be treated as transparent, while those "inside" a line are treated as opaque. "Outside" spaces are those occurring before/after the first/last non-space character. If you want all interior spaces to be treated as transparent too, toggle the "Transparent Block Paste" setting in the settings panel.

If you think the background on Layerboxes is too opaque to judge its exact target location, there is a "High Transparency Layerbox" setting. This will increase the displayed background transparency and change the Layerbox content's color to red with a faint cyan "halo" for better contrast.

● Clicking on a Layerbox focuses it. A focused Layerbox can be moved with arrow keys, deleted with Escape, or pasted with Enter.

● Creating a Layerbox adds it to the Layerboxes sidepanel. Every Layerbox entry in the sidepanel has a button to focus the associated Layerbox or delete the associated Layerbox. The list entries also display a tooltip of the Layerboxs' contents. Reorder list entries by drag-and-dropping them to reorder the order of Layerboxes in the document.

There is a chance that Layerboxes might desync with their intended document position in very large documents. This is untested.

※Paste Mode

There are three possible Paste Mode options. You can see the current one displayed in the status bar. Clicking the Paste Mode status bar entry or pressing Ctrl-Space will toggle between them.

Block Pasting Disabled

Pasting with Ctrl-V or right click will paste normally, as it would in any other text editor.

Block Pasting Enabled

Pasting with Ctrl-V or right click will directly composite the AA from your clipboard into the document. The document will temporarily enter a read only mode. You can adjust the precise location your clipboard contents will be composited into with Arrow Keys, cancel the operation with Escape, or confirm it with Enter.

Block Pasting Layerbox

Pasting with Ctrl-V or right click will insert your clipboard contents into the editor as a Layerbox.

※Extraction:

Extraction Copy [Ctrl-Alt-C]

The editor allows you to have multiple selections at a time. If you want to extract a section of AA out of a larger work without manually replicating the padding, select it and run Extraction Copy. This will extract the section of AA from the original, add extra padding so that the alignment is not disturbed, and place it on your clipboard.

Extraction Cut [Ctrl-Alt-X]

The same as above, but also replaces your selected AA with blank padding of equivalent size.

※Formatting:

All editor functions marked as a line operation will execute either for a set of selected lines or for every line in the document if the selection is empty (ie. there is only one or more active carets).

A caret at the start of a line at the end of a selection is still considered as extending the selection to the line it is on. This may be changed in the future.

Add Guideline to Right Side line operation [Ctrl-Alt-R]

Inserts a vertical guideline of "|" characters on the right side of your AA. The resulting guideline is always perfectly aligned regardless of if you have Unicode space generation enabled or disabled. When Unicode space generation is disabled, period padding is used to align the guideline. Add a guideline before you start making edits on the left side of your AA. By looking at how the guideline shifts from being perfectly aligned, you can keep track of to what extent your edits have misaligned original formatting.

When you're ready to remove the guideline, use the Remove Last Non-Whitespace Char to remove the guideline (if Unicode space gen is disabled, you might have to run it multiple times to remove period padding) and Remove Trailing Whitespace to remove the spacing it added.

Add FWS (11px) at Start line operation [Ctrl-I]

Adds a single FWS character at the start of the line. Does not regenerate the initial run of spaces.

Remove FWS (11px) at Start line operation [Ctrl-U]

Removes one FWS (11px) worth of space at the start of the line and regenerates the initial run of spaces.

Remove Trailing Whitespace line operation [Ctrl-G]

Removes all FWS, HWS, and Unicode space characters located at the end of a line.

Remove Last Non-Whitespace Char line operation [Shift-Ctrl-G]

Removes that last character that is not a FWS, HWS, or Unicode space.

Mirror Horizontally by Line line operation

Mirrors line(s) horizontally, replacing characters with horizontally mirrored equivalents when possible. May cause misalignment due to characters' mirrored matches not being the same width.

Mirror Horizontally by Selection

Mirrors each selection horizontally, independently from each other. Otherwise the same as Mirror Horizontally by Line.

Useful with rectangular selection.

Mirror Vertically line operation

Mirrors line(s) vertically, replacing characters with vertically mirrored equivalents when possible. May cause misalignment due to characters' mirrored matches not being the same width.

If you have selected a rectangular region, running this will only vertically mirror the selected area instead.

Move to Right Margin line operation

Shifts a line to the right margin. Will use period padding if Unicode space generation is disabled. Right margin can be changed in the Settings panel. Defaults to 790px.

★Add 1px SP at Caret, Remove 1px SP at Caret [Ctrl-RightArrow, Ctrl-LeftArrow]

Increase or decrease the space where the caret is located by one pixel at a time. The main function for adjusting padding. You will be using this often.

Add 1px SP at Start, Remove 1px SP at Start line operation [Shift-Ctrl-RightArrow, Shift-Ctrl-LeftArrow]

Increase or decrease the space at the start of each line by 1px.

Narrow All 1px from Caret to Left

Takes the distance of the caret from the left, finds the closest position to that distance on each line (if there is no exact match, goes to the closest position from the left), and removes 1px of space from each line at that position. If it is not possible to remove 1px space, a non-space character is removed and replaced with a run of spaces 1px less than the original character width.

Expand All 1px from Caret to Right

Takes the distance of the caret from the left, finds the closest position to that distance on each line (if there is no exact match, goes to the closest position from the left), and adds 1px of space to each line at that position. This and "Narrow All 1px from Caret to Left", in my opinion, are not very useful functions. I only implemented them because OrinrinEditor has them. You can simulate this exact behavior in more steps but with more precision using rectangle select, extraction cut, and layerbox compositing.

※Auto Adjust:

Set Reference Line Length [Ctrl-Shift-E]

Records the length of the line your caret is on. You can see the current Auto Adjust length in the status bar at the bottom of the editor.

Attempt Misalignment Adjustment [Ctrl-E]

Will attempt to modify the length of the current run of spaces your caret is positioned in so that the total length of the line matches the Auto Adjust length set as reference. Will use period padding if Unicode space gen is disabled. Will fail and do nothing if there are no spaces at the caret, if it was unable to generate the necessary length, or if there is not enough space to carry out the adjustment. Can be run on multiple lines at once, but only if there is only one caret on each line. If there are multiple carets on one line, all but the first will be ignored.

※Framedrawing:

MittensEdit contains functions for automatically inserting an AA frame. It comes with 20 styles of frames baked in, including speech bubbles. Please note that not all styles of frames will always generate perfectly, some manual adjustment may be required after generation.

Create Framebox

Inserts a special layerbox for drawing frames into your document.

Draw Frame Around AA line operation

Draws a frame around the selected lines of AA, or if there's no selection, around the entire document.

Framedrawing might not always properly work when Unicode space gen is disabled. This will be fixed in future updates.

※Brush Modes

When you open the "Brushes" panel in the sidebar, you enter a brush mode. These replace text in your AA based on mouse actions.
There are four brush modes. These can be switched using the dropdown menu in the brushes panel and are obnoxiously indicated with emoji.
The active brush text can be seen in the input field by the brush selection menu. This text can be set from the options in the sidebar or entered manually.

🖌️ Overwrite Spaces

Select a region of the document. When you release your mouse, the spaces in the selection will be replaced with the brush text. Orginal formatting is attempted to be preserved.

🪣 Overwrite All

Select a region of the document. When you release your mouse, all text in the selection will be replaced with the brush text. Orginal lengths are attempted to be preserved.

⚪ Eraser

Select a region of the document. When you release your mouse, all text in the selection will be replaced with empty padding space. Orginal lengths are attempted to be preserved.

✏️ Free Draw

Selections are disabled. When you click down anywhere on the document, the active brush text is pasted to that location. Move your mouse while holding it down to draw.

※Space Management:

These functions should be used carefully.

Manual Unicode Space Insertion Menu

A dropdown menu for manually inserting acceptable Unicode spaces of a certain pixel width.

Regenerate All Spacing line operation

Regenerates every run of spaces in a line as if they had been freshly inserted by an editor command.

Initial HWS to Unicode line operation

Space generation will normally use a HWS for 5px spacing, which causes formatting problems when at the start of a line. Converts every HWS at the start of a line to a 5px Unicode space. Does not attempt to regenerate any other spacing.

All Spaces to Pure SJIS with Period Padding line operation

If a run of spaces contains a Unicode space or bad spacing, attempts to replace it with only HWSs, FWSs, and periods. Will not always generate visually optimal spacing for small padding and will cluster periods together. Take this as a starting point, always try to spread your period padding out to be visually unobtrusive.

※Template Insertion

MittensEdit comes with a collection of built-in single-line and multi-line AA templates, consisting of commonly used characters and components.

For multi-line templates, using the drop-down menu, you can choose what happens when you select an entry. They can be inserted as layerboxes, inserted directly, or copied to the clipboard without inserting.

※Trace Mode

Displays an image in the background for use as reference. Features like this are how all those anime girl AA are made. Should be self-explanatory. Reset button resets image position and color to defaults.

Accessed by a panel in the sidebar

※Settings

Currently the settings panel contains only four entries.

Right Margin (px)
- The value in pixels used when aligning to right margin. Defaults to 790px.
Transparent Block Paste
- Treat all interior spaces in a Layerbox as transparent when compositing. Off by default.
Delete Layerbox After Paste
- Decides if Layerboxes are deleted after pasting. On by default.
High Transparency Layerbox
- Toggle alternative visual style for Layerboxes. Off by default.

※View

Preview AA [Ctrl-B]

Displays a preview of the current AA. Incorrect HWS will collapse as they would when posted. Any layerboxes you have open will display as if they'd been pasted down in their current locations.

TIPS:

  • Take notes on how other AA craftsmen achieved the sorts of curves and angles that they did, and what characters they frequently used for it. If some combination of characters is very useful, post about it on AA Bar and we might add it to the templates menu.
  • Leave Unicode space generation enabled while pasting layerboxes and performing 1px adjustment operations to avoid misalignment.

  • When attempting to eliminate Unicode space use, disable Unicode space generation and then attempt adjusting your run of padding space back and forth by nudging it with Ctrl-LeftArrow & Ctrl-RightArrow. This will sometimes get you the right width right away!

    This is because, when Unicode space gen is enabled, a run of spaces will generate with only S_JIS spaces only when the number of HWS needed is less than the number of FWS. If reaching that pixel width with only FWS and HWS while that condition is met is not possible, it will add Unicode spaces to fill the gap. This is so that it will never generate a HWS at the start of a run of spaces (unless there is only one HWS), but sometimes that extra HWS at the start of a run of padding is what's needed to achieve a specific width. Toggle Unicode generation frequently while working.

  • Be careful when running the Regenerate All Spacing function, it can regenerate runs of spaces that are already okay and add Unicode spaces to them! It's there to assist with cleanup, not handle all spacing for you.
  • Add periods (.) to your padding space to reach widths that normal use of FWS and HWS cannot. The editor has a built-in operation for this, "All Spaces to Pure SJIS with Period Padding," but it does not always generate visually optimal spacing, may add unneeded periods, and will cluster periods together. Use it as a starting point before making manual edits. Try to space out the periods you use to make them less visually intrusive! Look at the work of past AA craftsmen to see how they do this.
